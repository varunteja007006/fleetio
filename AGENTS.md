## Architecture

### Expo Router routes

File-based under `src/app/`:

| Route | Screen |
|-------|--------|
| `/` | Login or dashboard link |
| `/dashboard` | Tab navigator (Dashboard, Routes, Profile), edit-profile |
| `/admin` | Admin dashboard, route CRUD, assignments, user management |

`experiments.typedRoutes: true` and `experiments.reactCompiler: true` are on in `app.json`. URL scheme: `fleetio://`.

### Convex backend

- **Schema** (`convex/schema.ts`): 9 tables — `profiles`, `routes`, `checkpoints`, `routeAssignments`, `routeRuns`, `incidents`, `alerts`, `whatsappGroups`, `routeWhatsappMappings`.
- **Auth flow**: Uses `@convex-dev/better-auth` component (`convex/convex.config.ts` mounts it). `convex/auth.ts` creates the Better Auth server with phone OTP plugin. `convex/http.ts` registers auth routes via `authComponent.registerRoutes()`. Client-side auth in `src/lib/auth-client.ts` uses `@better-auth/expo` + `expo-secure-store`. Email/password sign-up is also enabled alongside phone OTP.
- **Auth config**: `convex/auth.config.ts` uses `getAuthConfigProvider()` from `@convex-dev/better-auth` (not a standard JWT provider). `tokenIdentifier` is derived from the component API, not from `ctx.auth.getUserIdentity()`.
- **Current user lookup**: Use `authComponent.getAuthUser(ctx)` from `convex/auth.ts`, not `ctx.auth.getUserIdentity()`.
- **OTP in dev**: Stubbed — logs code to console. Production SMS sending is commented out in `convex/auth.ts`.
- **Soft-delete**: `isDeleted: boolean` on profiles, routes, and checkpoints. Never hard-delete these tables. Other tables use hard deletes.
- **Role system**: `admin` / `manager` / `driver` / `new_user` with `pending` / `approved` / `rejected` status. New users default to `new_user` + `pending`.

### Convex rules (condensed; see `.github/convex.instructions.md`)

- Always include argument validators on all functions.
- **Don't** use `.filter()` — define indexes and use `.withIndex()`. **Don't** use `.collect()` unbounded — use `.take()` or paginate. **Don't** use `.collect().length` — maintain a denormalized counter.
- Existing code has violations (`checkpoints.ts` uses `.take(10000)` without index + `.filter()`, `routes.ts` counts via `filter().length`). Do **not** copy these patterns.
- Index naming: `"by_field1_and_field2"` (include all fields).
- Use `ctx.db.patch()` for partial updates, `ctx.db.replace()` for full replaces.
- Actions cannot access `ctx.db` — use `ctx.runQuery`/`ctx.runMutation`.
- Files using Node.js APIs need `"use node";` at top.
- `tokenIdentifier` is the canonical user identifier (not `subject`).
- Env vars in Convex: use `env` from `_generated/server`, not `process.env`.
- `undefined` is not a valid Convex value — use `null` instead.

## Forms

TanStack React Form with Zod v4 validation. The custom wiring lives in:
- `src/components/forms/contexts.ts` — `createFormHookContexts()`
- `src/components/forms/hooks.ts` — `createFormHook()` exports `useAppForm`
- `src/components/forms/form-input.tsx` — reusable `FormInput` field component
- `src/components/forms/form-base.tsx` — shared label/error wrapper

Usage pattern: `form.AppField` + `<field.Input label="..." />`.
